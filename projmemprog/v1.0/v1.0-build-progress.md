## 4. Architecture Decisions

### 4.1 Git-Optional Scoring
**Decision:** `StaticScorer` works everywhere; `GitScorer` adds git history when available.
**Rationale:** Graceful degradation in CI, shallow clones, non-git directories.
**Location:** `engine/scoring.py`

### 4.2 Security Redaction
**Decision:** Default to redaction (`[REDACTED]`), with `--security-warn` for warn-only mode.
**Rationale:** Security-first design. Enterprise procurement requirement.
**Location:** `engine/security.py`

### 4.3 Budget Heuristic
**Decision:** Ratio-based: >2.0→strip, >1.5→signatures, ≤1.5→user mode.
**Rationale:** Simple, predictable, never drops files.
**Location:** `engine/pruner.py` `_resolve_effective_mode()`

### 4.4 MCP as Optional Dependency
**Decision:** `mcp` package installed via `pip install codebase-shaker[mcp]`.
**Rationale:** `mcp` package is in Beta. Keeps core tool stable.
**Location:** `pyproject.toml` `[project.optional-dependencies]`

### 4.5 Signal Handling for Remote Cleanup
**Decision:** Register SIGINT/SIGTERM handlers in `clone_remote()`, restore defaults in `cleanup_remote()`.
**Rationale:** Ensures temp dir cleanup even on Ctrl+C.
**Location:** `engine/remote.py`

## 8. Lessons Learned

1. **Function name shadowing:** The `score_files` function import was shadowed by the `score_files` boolean CLI parameter. Fixed with underscore alias (`_score_files`). Lesson: avoid naming collision between imports and CLI params.

2. **Splitlines vs splitlines(keepends=True):** `str.splitlines()` + `"\n".join()` drops trailing newlines. Detected by unit test. Fixed by detecting trailing newline separately.

3. **FrozenInstanceError:** `SecurityFinding` is `frozen=True`, couldn't set `finding.redacted = True`. Fixed by tracking `redacted_count` as a separate counter.

4. **Test output vs stdout:** Click's `CliRunner` mixes stdout and stderr in `result.output`. JSON parsing failed because of stats lines. Fixed by using file output (`-o`) in integration tests.

5. **CLI test isolation:** `monkeypatch.chdir()` in one test affected subsequent tests that relied on CWD. Tests must be independent and clean up after themselves.

6. **Config field validation:** `_comment` in `.shakerrc.json` template caused config validation to reject it. Removed from template. Lesson: keep config templates clean.
